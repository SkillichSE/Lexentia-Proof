How ModelArena Works

A GitHub Actions workflow runs every morning, fires 180 prompts across 15 free AI models, and publishes the results here. No paid APIs. No manual work.

Why automate this? Model quality changes constantly — providers deprecate models, release updates, and tweak inference settings without notice. A daily benchmark catches regressions and improvements as they happen, building a historical record that one-off evaluations can't provide.

🎯 Scoring Philosophy

Most benchmarks collapse everything into one score — which means a tiny 8B model that runs at 600 tok/s will always lose to a 70B that generates 40 tok/s but writes better code. That's not a useful comparison. ModelArena keeps speed and quality completely separate, and groups models by size tier.

💻
Code
40% of Quality
Three Python tasks — prime check, binary search, LRU cache. Scored on function structure, logic, and documentation.
🧠
Reasoning
40% of Quality
Logic syllogism, speed math, 12-ball puzzle. Answers matched against known correct solutions.
🌍
Translation
20% of Quality
EN↔RU and EN→ES. Verified by script detection and target-language keyword matching.
⚡
Speed
Separate board
Tokens per second across three prompt lengths. Normalised to 100 within each run — never mixed with quality.

Size categories — models are grouped as small (≤10B), medium (10–50B), and large (50B+). Rankings within a tier are the most meaningful comparisons. A small model that beats other smalls is genuinely impressive; ranking it against a 671B MoE is not.

🚀 Run It Yourself

Fork the repository
Fork ModelArena on GitHub — the workflow and all code comes with it.
Enable GitHub Pages
Settings → Pages → Source: main branch → Folder: /docs
Add API keys to Secrets
Settings → Secrets → Actions. Add GROQ_API_KEY (required), GOOGLE_API_KEY (optional), OPENROUTER_API_KEY (optional). Google and OpenRouter keys are optional — Groq alone gives you 5 models.
Trigger the first run
Actions → Daily AI Benchmark → Run workflow. Takes ~20 minutes. Results appear automatically.

➕ Adding Models

Edit src/config.py and add an entry to the appropriate provider section. For OpenRouter, always use the :free suffix — models without it require credits.

# src/config.py → "openrouter" section "my-model": { "id": "provider/model-name:free", "name": "Display Name", "provider": "OpenRouter", "size": "7B", "context": "128k" }
